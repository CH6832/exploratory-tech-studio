
3. Class Diagram

Purpose: To show the static structure of the system with detailed classes and relationships.

    Classes:
        User: Attributes like userId, methods like login().
        Heatmap: Attributes like data, methods like generateHeatmap().
        InputForm: Attributes like formData, methods like submit().
        Database: Attributes like historicalInputs, methods for CRUD operations.
